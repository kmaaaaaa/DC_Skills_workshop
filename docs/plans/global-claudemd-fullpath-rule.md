# 計画: グローバル CLAUDE.md に「MD/HTML 提示時はフルパス表示」ルールを追加

## 1. 背景・目的

ユーザー要望:
> グローバル claude.md に、markdown と HTML ファイルを提示する際には相対パスではなく必ずフルパスを表示するように強制してください

現状、計画書・要件書・設計書・レポート等の MD/HTML を生成した際、応答内で `.tmp/plan-foo.md` のような **相対パス** で提示してしまうことがある。
ユーザーは Windows のエクスプローラやエディタから直接開きたいため、`C:\Users\KosukeMaeda\...` のような **フルパス（絶対パス）** で提示することを強制したい。

対象ファイル:
- `C:\Users\KosukeMaeda\.claude\CLAUDE.md`（グローバル）

## 2. 変更内容（最小変更）

### 追加場所

`## ドキュメント運用` 直下に、新規サブセクション **`### MD/HTML ファイル提示時の絶対パス表示（必須）`** を追加する。

理由:
- 既存の `### Markdown 成果物の視認性確保（HTML同時生成）` と密接に関連（生成した MD/HTML をユーザーに渡す出口ルール）
- ドキュメント運用セクション内で「生成」→「視認性（HTML ペア）」→「提示時のパス表示」と論理的に並ぶ
- 他セクション（最上位ルール等）を肥大化させない

### 追加するテキスト案（最終版）

```markdown
### MD/HTML ファイル提示時の絶対パス表示（必須）

ユーザーに対して Markdown ファイル（`.md`）または HTML ファイル（`.html`）の保存先・参照先を提示する際は、**必ずフルパス（絶対パス）で表示する**。相対パスのみでの提示は禁止する。

#### 対象

- 新規生成した計画書・要件書・設計書・レポートのパス提示
- 既存 MD/HTML への参照案内（「○○.md を参照してください」等）
- HTML ペア生成完了時の保存先報告
- ユーザーへの完了報告に含まれる MD/HTML パス全般

#### 表示形式

- Windows: `C:\Users\KosukeMaeda\...\plan-foo.md` 形式（バックスラッシュ区切り）
- 文中・箇条書き・コードブロック内のいずれでも絶対パスを使用する
- ファイル名のみへの言及（パスを示さない単なる引用）は対象外

#### 例

- ❌ NG: `.tmp/plan-foo.md に保存しました`
- ❌ NG: `docs/plans/foo.md を参照してください`
- ✅ OK: `C:\Users\KosukeMaeda\Documents\project\.tmp\plan-foo.md に保存しました`
- ✅ OK: `C:\Users\KosukeMaeda\Documents\project\docs\plans\foo.md を参照してください`

#### 対象外（相対パスのまま OK）

- コミットメッセージ・PR 説明文（リポジトリ内パスが慣習）
- ソースコード内のパス文字列（`import` 文・設定値等）
- `file_path:line_number` 形式のコード位置参照（IDE が解決するため相対で十分）
- CLAUDE.md・README.md 内のサンプル記述（ドキュメント執筆コンテキスト）
```

## 3. 実装手順

1. `C:\Users\KosukeMaeda\.claude\CLAUDE.md` の `## ドキュメント運用` セクション内、`### ドキュメント更新の原則` と `### Markdown成果物の視認性確保（HTML同時生成）` の **間** に新サブセクションを挿入する
   - 順序: ドキュメント更新の原則 → **MD/HTML ファイル提示時の絶対パス表示（必須）** → Markdown 成果物の視認性確保 → ...
2. 編集後、Read で実ファイル内容を検証する（CLAUDE.md「Write/Edit後の検証ルール」遵守）
3. ユーザーに完了報告（追加位置・行数を絶対パスで提示）

## 5. Devil's Advocate

### 指摘 1: 「フルパス強制」は過剰ではないか？

- **指摘**: 全ての MD/HTML 言及をフルパスにすると、可読性が低下する箇所がある（例: 文中で何度も `C:\Users\KosukeMaeda\...\foo.md` を繰り返すと冗長）
- **対応**: ルール文に「ファイル名のみへの言及（パスを示さない単なる引用）は対象外」と明記。1 度フルパスを提示した後、同じ応答内では `foo.md` のような短縮形に切り替え可能とする運用を許容する形にした
- **追加対応**: 「対象外」リストに `file_path:line_number` 形式やコード内パスも明記し、誤適用を防止

### 指摘 2: CLAUDE.md 内のサンプル記述まで書き換える必要があるか？

- **指摘**: ルール内のサンプルコード（`example: docs/plans/foo.md` 等）まで絶対パス化すると、ドキュメントとして読みづらい
- **対応**: 「CLAUDE.md・README.md 内のサンプル記述（ドキュメント執筆コンテキスト）」を対象外に明記

### 指摘 3: クロスプラットフォーム互換性

- **指摘**: 「Windows: バックスラッシュ区切り」と固定すると、将来 macOS/Linux で使用する場合に矛盾する
- **対応**: 現状のユーザー環境は Windows 確定（既存 CLAUDE.md にも Windows 環境ルールが既述）。将来移行時はそのタイミングで更新する判断とし、グローバル CLAUDE.md は Windows 前提でよい

### 指摘 4: 「強制」と書いた場合の罰則・違反検出

- **指摘**: 「必須」「禁止」と書いても、実際に Claude が忘れた場合の検出メカニズムがない
- **対応**: 現状の他の必須ルール（「Write/Edit後は必ずReadで検証」等）も同様にメカニズム未整備。ユーザー側の指摘で都度修正する運用で運用実績がある。フック導入は別途検討する課題として留保

### 指摘 5: 最小変更原則違反のチェック

- **指摘**: 周辺セクションのリファクタや、既存ルールの整理も同時に行う誘惑がある
- **対応**: 今回は **1 セクション追加のみ**。既存セクションへの編集は行わない。完了報告時の改善提案も「別途確認」とする
